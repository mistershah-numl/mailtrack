  Response: Returns to browser
  Status: ✅ Works

VERCEL FLOW:
  Browser: https://your-app.vercel.app

  Database: Connected via DATABASE_URL (SAME!)
  Response: Returns to browser

                    WHAT HAPPENS WHEN USER SIGNS UP
================================================================================

1. User goes to: https://your-app.vercel.app/sign-up
2. Fills in: Name, Email, Password
3. Clicks: "Create Account"
4. Request sent to: /api/auth/sign-up (auto routes to Vercel!)
5. Server hashes password
6. Server creates user in database
7. Server creates session
8. Browser gets session cookie
9. User is logged in!
10. Dashboard loads
11. User sees dashboard
12. User can now install extension!

All automatic! No manual steps!

================================================================================
                    WHAT HAPPENS WHEN USER TRACKS EMAIL
================================================================================

1. User in Gmail clicks: "Track This Email" button (from extension)
2. Extension adds tracking pixel to email body
3. Generates unique tracking ID
4. User sends email normally
5. Extension sends: POST /api/emails/track (auto routes to Vercel!)
6. Server creates email record in database
7. Server returns success
8. Extension confirms: "✓ Tracking Enabled"
9. User sees email in dashboard
10. Recipient opens email
11. Tracking pixel loads
12. Request to: /api/track/[trackingId] (auto routes!)
13. Server records open in database
14. Dashboard updates immediately!
15. User sees: "Opened: 2 hours ago, Device: iPhone, Browser: Apple Mail"

All automatic! Real-time updates!
